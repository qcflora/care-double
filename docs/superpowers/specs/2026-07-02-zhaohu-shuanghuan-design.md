# 「照护双环」产品设计规格

> 嵌入式照护者心理支持与照护智慧协同小程序

**版本**：v1.0  
**日期**：2026-07-02  
**状态**：设计确认，待实现

---

## 1. 产品概述

### 1.1 核心问题
家庭照护者在长期照顾高龄老人时，面临双重崩溃风险——照护者的心理先于老人身体崩溃。现有方案存在三个根本缺陷：
1. **"额外负担"悖论**：心理调节课程要求照护者挤出时间学习，本身就在增加负担
2. **知识碎片化**：WHO标准、国家"九防"规范散落于论文中，普通家庭找不到、看不懂
3. **多人照护信息黑洞**：子女和护工交接时用药遗漏、翻身记录断层频发

### 1.2 产品定位
一款可交互的小程序，将心理调节嵌入照护间隙，将专业知识转化为交互工具，将单人照护升级为多人协同。同时支持生成纸质手册、表格，并通过语音播报等格式，适配不同文化水平的照护者。

### 1.3 「双环」模型
- **内环（心理支持环）**：照护者的情绪蓄水池——在照护动作的间隙，自动嵌入微调节，让照护老人的每一个停顿，都成为照护自己的机会
- **外环（智慧协同环）**：照护行动的神经网络——把散落的照护知识、多人协作、应急响应织成一张网，让每个人都不是孤军奋战

## 2. 目标用户与场景

### 2.1 用户类型
| 用户类型 | 年龄 | 特征 | 核心需求 |
|---------|------|------|---------|
| 核心照护者 | 50-70岁子女 | 主要照护提供者，时间长、压力大 | 心理支持、专业指导、减轻负担 |
| 协助照护者 | 亲属、邻里 | 不定期提供帮助 | 快速了解老人状况、知道该做什么 |
| 远程子女 | 外地工作 | 不在身边，心有余力不足 | 掌握老人情况、远程参与协同 |

### 2.2 使用场景
- 日常照护语音引导
- 连续照护后情绪崩溃时的自动干预
- 多人交接班信息同步

- 月度家庭健康评估会议

## 3. 核心设计原则

1. **零额外负担**：不要求用户"学习"，所有心理调节和知识获取都嵌入现有照护流程的间隙
2. **多模态适配**：同一个内容，自动提供语音播报、图文卡片、可打印纸质版三种形式
3. **温暖克制**：不贩卖焦虑，不强调"病患"，而是"陪伴"与"支持"的语气
4. **适老化设计**：大字体、高对比度、简单操作路径、语音优先

## 5. 核心页面设计

### 5.1 启动引导页
- 柔和渐变暖色调背景
- Slogan：「照护双环 · 陪你一起照顾最爱的人」
- 身份选择：核心照护者 / 协助照护者 / 远程子女 / 专业护工
- 选择后自动适配界面内容和操作路径

### 5.2 今日照护（首页）
**顶部区域**
- 问候语 + 日期
- 照护负荷指示条（绿/黄/红三级）
- 负荷过高时自动提示「你今天已经很辛苦了，要不要休息一下？」

**今日任务列表**
- 卡片式任务，按时段排序（早/中/晚/夜）
- 任务类型：用药、翻身、进食、清洁、测量、散步等
- 每张卡片右侧「语音引导」按钮
- 已完成任务显示绿色勾选

**照护间隙模块**
- 完成一个任务后，自动弹出卡片：「刚刚辛苦了，要不要花3分钟呼吸一下？」
- 提供三种调节模式快速选择

**底部悬浮**
- 应急快速入口按钮

### 5.3 语音引导照护页
- 全屏沉浸式，大字号步骤卡片
- 当前步骤高亮，其余步骤半透明
- 语音自动播报（可暂停/重播/上一步/下一步）
- 关键注意事项用橙色卡片强调
- 进度条显示当前进度
- 完成后自动记录并询问「完成情况如何？」（良好/一般/需关注 三选一）

### 5.4 嵌入式心理调节页
**三种模式**
1. **呼吸调节**：478呼吸法，环形动效跟随呼吸节奏缩放
2. **身体扫描**：引导式，从头到脚逐步放松
3. **情绪释放**：语音倾诉/文字书写，AI共情回应

**通用设计**
- 柔和背景渐变
- 环形进度动效
- 结束后显示鼓励语 + 自动返回照护流程
- 「我想多待一会儿」选项

### 5.5 照护圈（多人协同）
- 成员头像环形排列（「双环」视觉隐喻）
- 当前值班人高亮标识，带"值班中"标签
- 交接班看板：待办事项、用药记录、老人状态、注意事项
- 「一键交班」大按钮
- 交班时自动整理未完成事项 + 异常情况
- 交接记录时间线

### 5.6 交接班详情页
- 时间线式记录
- 四大模块：用药、饮食、情绪、身体状况
- 异常项红色标注
- 支持语音记录交班信息，自动转文字
- 接班人确认按钮

### 5.7 应急中心
- 卡片式急救场景入口：噎食、跌倒、心脏骤停、烫伤、走失

### 5.8 急救指导页

### 5.9 健康档案

## 6. 关键交互流程

### 6.1 嵌入式心理调节（核心创新点）
```
完成照护任务
  → 弹出卡片：「刚刚辛苦了，要不要花3分钟呼吸一下？」
  → 用户点击「好」
    → 进入呼吸调节页（环形动效 + 语音引导）
    → 3分钟后
      → 「感觉好点了吗？」
      → 返回照护流程
  → 今日心理支持+1，负荷指示条下降
```

### 6.2 语音引导照护
```
点击任务「帮老人翻身」
  → 进入语音引导页
  → 语音自动播报第一步
  → 大字号文字同步显示
  → 用户完成后点「下一步」
  → 直到全部完成
  → 「记录一下老人的状态？」→ 快速选择（良好/一般/需关注）
  → 自动同步到照护圈
```

### 6.3 多人交接班
```
当前值班人点「交班」
  → 自动整理未完成事项 + 异常情况
  → 生成交接单
  → 下一位值班人接收 → 确认接班
  → 所有人收到通知
  → 交接记录永久可查
```

### 6.4 应急急救
```
噎食/跌倒等紧急情况发生
  → 打开应急中心
  → 选择对应场景
  → 进入急救指导页
  → 视频 + 语音分步指导
  → 同时自动通知紧急联系人
  → 黄金救援时间倒计时
  → 「我已拨打120」状态确认
```

## 10. TRAE 技术融合方案

本产品的核心竞争力在于将 TRAE 的 AI 能力深度嵌入照护全流程，而非作为"概念点缀"。以下三个核心链路均接入真实 LLM 能力，通过结构化 Prompt + RAG 知识库实现。

### 10.1 链路一：动态知识 RAG 语音引导

**痛点**：现有照护步骤是静态写死的，无法根据老人的基础疾病（高血压、糖尿病、骨质疏松等）和当天状态动态调整。

**TRAE 能力运用**：将 WHO 照护标准、国家"九防"规范建为 RAG 知识库，TRAE 根据老人档案 + 当天状态，动态生成个性化语音引导脚本。

**数据流**：
```
老人档案（疾病/用药/过敏）
  + 当天状态（血压/血糖/情绪评分）
  + 任务类型（翻身/喂药/进食）
    → RAG 检索相关知识片段
    → TRAE LLM 生成个性化引导脚本
    → 输出：步骤化 JSON + 语音播报文本
```

**Prompt 设计**：
```
系统角色：你是一位资深养老护理专家，精通WHO照护标准和国家"九防"规范。

用户输入：
- 老人档案：82岁女性，高血压、2型糖尿病、轻度骨质疏松
- 当前用药：氨氯地平（降压）、二甲双胍（降糖）、阿司匹林
- 当天状态：血压135/85，血糖8.2（略高），情绪低落
- 任务类型：协助翻身

请生成5步以内的翻身指导脚本，要求：
1. 每步包含标题、操作描述、注意事项
2. 针对骨质疏松：强调动作轻柔，避免关节拉扯
3. 针对高血压：提醒翻身速度要慢，防止体位性低血压
4. 语言通俗，适合50-70岁照护者理解
5. 输出为JSON格式：[{title, description, tip}]

RAG 检索上下文：
[WHO长期照护指南-翻身护理章节] 翻身频率每2小时一次...
[九防规范-防压疮] 骨突部位需悬空减压...
```

**输出示例**：
```json

    "tip": "李奶奶有骨质疏松，翻身时动作要特别轻柔，避免拉扯手臂关节"
  },
  {
    "title": "移动双腿",
    "description": "将老人双膝轻轻弯曲，双脚平放床上。一手托住膝部，一手扶住脚踝。",
    "tip": "骨质疏松患者关节脆弱，切勿用力拉扯膝关节"
  },
```

### 10.2 链路二：语音交接班自动摘要 + 异常检测

**痛点**：照护者（尤其50-70岁）打字困难，交接信息靠微信群口头描述，信息遗漏频发。

**TRAE 能力运用**：照护者语音交班 → TRAE 语音转文字 → 自动提取用药/饮食/情绪/身体四大模块摘要 → 异常关键词自动标红 → 生成交接单。

**数据流**：
```
照护者语音输入（自然语言）
  → TRAE ASR 转文字
  → TRAE LLM 结构化提取
    → 用药模块摘要 + 异常标记
    → 饮食模块摘要 + 异常标记
    → 情绪模块摘要 + 异常标记
    → 身体模块摘要 + 异常标记
  → 自动生成结构化交接单
  → 推送到照护圈所有人
```

**Prompt 设计**：
```
系统角色：你是一位养老护理交接班记录员，请将照护者的口语化交接内容提取为结构化交接单。

用户语音转文字：
"嗯，今天上午那个降压药已经吃了，血压量了一下135到85，还算正常。
早餐吃的不太好，就喝了半碗粥，一个鸡蛋都没吃完，说没什么胃口。
下午的时候老太太情绪有点低落，一直说想儿子了，我陪她聊了一会儿。
然后下午三点翻了一次身，皮肤还好没有发红。
对了，血糖量了一下8.2，比正常高了一点，要注意。"

请提取为以下JSON格式：
{
  "medicationNotes": "用药情况摘要",
  "dietNotes": "饮食情况摘要",
  "moodNotes": "情绪状态摘要",
  "healthNotes": "身体状况摘要",
  "abnormalItems": ["异常项1", "异常项2"],
  "pendingTasks": ["待办任务"]
}

异常判定规则：
- 血糖 > 7.0 mmol/L 标记为异常
- 血压 > 140/90 标记为异常
- 食欲明显下降标记为异常
- 情绪低落持续标记为异常
- 皮肤发红/破损标记为异常
```

**输出示例**：
```json
{
  "medicationNotes": "晨间降压药已服用，血压135/85，正常",
  "dietNotes": "早餐食欲不佳，仅半碗粥，鸡蛋未吃完",
  "moodNotes": "下午情绪低落，想儿子，已陪伴聊天疏导",
  "healthNotes": "下午3点已翻身，皮肤无异常。血糖8.2mmol/L",
```

**Demo 体现**：交接班页面的"语音交班"按钮可录音，录音后调用 TRAE 进行转文字+结构化提取，自动填充交接单各模块。异常项自动红色高亮。

### 10.3 链路三：同源多模态内容生成

**痛点**：同一份照护知识，不同文化水平的照护者需要不同形式（语音/图文/纸质），传统方式需要分别制作三套内容。

**TRAE 能力运用**：同一份源知识输入 TRAE，一次性生成语音播报脚本、分步骤图文卡片描述、A4可打印排版（Markdown）。

**数据流**：
```
源知识（如"翻身护理标准流程"）
  → TRAE LLM 多态生成
    → 语音版：口语化播报脚本（适合低识字率）
    → 图文版：5步分卡片描述 + 插图提示词
    → 打印版：A4排版 Markdown（含注意事项框）
  → 用户切换Tab即时查看不同形态
```

**Prompt 设计**：
```
系统角色：你是照护内容多模态生成引擎，请将同一份照护知识输出为三种形态。

源知识：
协助老人翻身的标准流程：
1. 准备：关门窗、调室温、掀盖被
2. 移腿：弯曲双膝，双脚平放
3. 侧身：扶肩扶髋，转向对侧
4. 垫枕：背部垫枕，腿间夹垫
5. 检查：查看皮肤，记录时间

请分别生成：

【语音版】适合低识字率照护者的口语化播报脚本，每步30秒以内，语气亲切温和。

【图文版】5张步骤卡片，每张含：步骤编号、标题（8字内）、操作描述（30字内）、插图提示词（用于AI生图）。

【打印版】A4纸排版格式，含标题、步骤详情、注意事项框、签字栏。输出为HTML/Markdown。

输出JSON：{ audio_script, image_cards[], print_layout }
```

**Demo 体现**：AI多模态适配演示页面，用户选择一个照护主题后，TRAE 一次性生成三种形态内容，Tab切换即可查看。语音版可真实播放 TTS。

## 13. 交互细节参数

### 13.1 呼吸动画参数
- **吸气阶段**：4秒，环形 scale(1) → scale(1.15)，ease-in-out
- **屏息阶段**：3秒，环形保持 scale(1.15)，无变化
- **呼气阶段**：8秒，环形 scale(1.15) → scale(0.95)，ease-in-out
- **完整周期**：15秒/轮，推荐 5 轮（总计 75 秒）
- **三层环动效**：外环延迟 0s，中环延迟 0.15s，内环延迟 0.3s，产生涟漪效果

### 13.2 语音引导状态机
| 状态 | 视觉表现 |
|-----|---------|
| 当前步骤 | 100% 不透明度，边框 #7BA394 2px，背景 #FFFFFF |
| 已完成步骤 | 50% 不透明度，文字 #8B8680，左侧绿色勾选 |
| 待完成步骤 | 30% 不透明度，文字 #B5B0A8 |
| 播放中 | 底部脉冲指示器，2px 圆点 #7BA394，animate-pulse 2s |

### 13.3 触控热区
- **标准模式**：所有按钮最小 44×44px
- **老年人模式**（大字体模式）：按钮最小 56×56px，间距增至 16px
- **底部Tab**：高度 72px，图标 22px + 文字 12px
- **悬浮按钮**：56×56px，距底部 92px

### 13.4 无障碍设计
- **语音播报语速**：3档可选 — 慢（0.8x）、中（1.0x）、快（1.2x）
- **色盲适配**：主色灰绿(#7BA394)与警示红(#D96B5C)在红绿色盲下区分困难，应急场景额外使用图标+文字标识（而非仅靠颜色）
- **对比度**：主文字 #3D3A36 on #FAF9F7 对比度 9.2:1，远超 WCAG AA 标准 4.5:1
- **字体放大**：老年人模式全局放大 1.2x，标题 29px、正文 20px

## 14. 参赛亮点提炼

1. **TRAE 深度融合**：三个核心链路（RAG语音引导、语音交接摘要、多模态生成）均接入真实 AI 能力，而非 Mock 概念

5. **AI多模态适配**：同源内容三态输出，让专业知识真正触达每一位照护者
6. **温暖治愈的视觉语言**：去医疗化，用温暖设计降低心理门槛
7. **市场空白切入**：竞品聚焦老人端，本产品聚焦照护者端，心理支持+协同是明确蓝海
8. **可跑通的闭环 Demo**：完整走通"照护→调节→交班"全流程，而非静态界面陈列
